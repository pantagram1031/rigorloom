# v0.11-Z5 — anti-fabrication frontier

Wave goal: close the citation/consistency blind spots that let a report *look*
correct while asserting things the sources, the saeteuk, or the units don't
actually support. All new gates default WARN unless the failure is a provable
integrity break (a hard, machine-checkable contradiction), matching the Z1–Z4
calibration philosophy: a false-blocking gate is worse than none.

Owner: orchestrator (Claude). Implementers: codex `-s workspace-write` rooted at
the rigorloom checkout. Review: sol adversarial + 1 opus judgment pass.

## Slices (ranked by risk × cheapness)

### S1 — check_sources (citation reality)  [HARD on provable-fake, WARN otherwise]
Problem: an agent can invent a DOI/ISBN or a plausible-looking journal cite that
grep-passes every existing gate. Nothing verifies the reference *exists*.
- Parse the report's reference/endnote list into structured entries
  (author, year, title, container, DOI, ISBN, URL).
- Offline-first: verify against a local cache under
  `<PROFILE_ROOT>/cache/sources/` (Crossref DOI records, ISBN records) that the
  operator populates out-of-band; NO network at gate time (deterministic,
  air-gappable). Cache miss => WARN `source_unverified`, never HARD.
- HARD only for *internal* impossibility that needs no network:
  malformed DOI syntax that cannot resolve, ISBN checksum failure,
  future publication year, a DOI/ISBN that the cache says maps to a
  DIFFERENT title than cited (provable mismatch).
- Never call an LLM "does this look real" — deterministic only.
- Wire into content_audit (4.5) as the 6th sub-checker.

### S2 — saeteuk ↔ report consistency  [HARD on numeric contradiction]
Problem: the 세특 (short record) and the body report can drift — a number or a
named claim in the saeteuk that the body contradicts is a submission risk.
- Given a workspace holding both a saeteuk artifact and the report body,
  extract numeric claims + named entities from the saeteuk.
- HARD when a saeteuk number has a same-context body number that contradicts it
  (same unit, same subject, different value beyond rounding).
- WARN when a saeteuk claim/entity has no supporting mention in the body
  (`saeteuk_unsupported`).
- If no saeteuk artifact present, checker no-ops (pass).

### S3 — unit / dimension consistency  [WARN]
Problem: numeric claims with mismatched or impossible units (e.g. a length
reported in seconds, a rate whose numerator/denominator units don't cancel).
- Lightweight unit tagger over numeric spans (SI + common report units).
- WARN `unit_mismatch` on same-quantity restatements with incompatible units,
  and `unit_impossible` on a dimensional nonsense pairing.
- Deterministic dictionary; no external unit library dependency at gate time.

### S4 (stretch, only if S1–S3 land clean) — bibliography FORMAT consistency [WARN]
- Detect mixed citation styles / inconsistent ordering in the reference list.
- WARN only; purely cosmetic-integrity.
