# v0.9 — Hancom-free document stack, Studio v2, edit workflow, conformance

Status: IN PROGRESS (2026-07-13). Follows v0.8. Operator directives: (1) the FULL
pipeline capability — including HWP-family assembly and a rendering proof — must
work on Linux with no Hancom install; (2) major Studio functionality overhaul;
(3) agents must actually follow the workflow, and EDITING an existing report
gets its own first-class workflow.

## Research verdict (2026-07-13, web-grounded)

- HWPX is a national-standard open format (KS X 6101 OWPML): zip + XML. Editing
  and assembly without Hancom is fully feasible; `python-hwpx` (pure Python,
  active) proves the ecosystem; our own trouble-table (T7–T14) already encodes
  the XML-level pitfalls.
- Equations live in the XML as HwpEqn script — the exact script our eqn.py
  emits. Insertion without Hancom is straightforward; on-screen rendering
  depends on the viewer.
- Rendering to PDF without Hancom: **H2Orestart** (GPL-3 LibreOffice import
  filter for hwp/hwpx, Debian-packaged as `libreoffice-h2orestart`, active
  releases) enables `soffice --headless --convert-to pdf` on Linux. Fidelity
  for equations/complex forms is undocumented → must be measured, and the tier
  is honestly labeled **advisory proof** (submit-grade visual proof remains a
  Hancom step where available).
- Rejected: online/commercial converters (privacy — student documents never
  leave the machine), writing a full OWPML renderer (excessive), hwpxlib (JVM
  dependency; LibreOffice already covers the render path).

## X1 — hwpx XML assembly backend (Hancom-free fill)

hwp-master side (the document adapter owns document mechanics):
- `scripts/xml_backend.py` — stdlib-only applier for build_report ops against a
  .hwpx: `edit --file form_copy.hwpx --ops ops.json --save-as out.hwpx`.
  Op coverage v1: goto_text anchor resolution (single-run substring — T6),
  insert_text with para breaks + charPr inheritance guards (T7/T8/T10: never
  drain blanks near headings, split before in-table label paras, force justify
  after split), insert_equation (hp:equation node carrying the HwpEqn script,
  treatAsChar for inline), insert_table (cellSz per col — S2 knowledge),
  bold segment runs, page_break_before. Unsupported ops fail loudly (exit 4,
  listed) — never silently skipped.
- Reuse existing offline infra: tidy_hwpx (cleanup), style_diff/charpr_check
  (verification), form_inspect (anchors). fill_report gains
  `--engine com|xml` (default com on Windows, xml elsewhere); the xml engine
  runs build → xml_backend edit → tidy → style_diff, and marks the PDF-proof
  phase `proof_unavailable` unless a renderer is configured.
- Renderer hook: `--pdf-cmd` argv template (e.g. soffice headless with
  H2Orestart) — optional, never required.
- Parity harness (fixture-gated like the other WS tests): same form + same ops
  through COM and XML engines → unzip both, compare section XML normalized
  (charPr/paraPr id-stable diff) + style_diff clean on both.

rigorloom side:
- doc_backend gains tier `hwpx` (4th): calls hwp-master fill_report --engine
  xml via configured argv (external-adapter pattern like hwp, but works on
  Linux); bundle stays the zero-dep floor.
- CI (ubuntu): install `libreoffice --no-install-recommends` + h2orestart deb,
  build a tiny synthetic hwpx via xml_backend, convert to PDF headless, assert
  non-empty text layer → proves the whole Hancom-free chain end-to-end.
  (Gated `continue-on-error: false` but isolated job so flakiness is visible.)

## X2 — Studio v2 (functionality overhaul)

- Multi-workspace dashboard: cards with stage progress, gate states, lint
  status, last event time; workspace switcher.
- Gate provenance panel: `.pipeline/gate_checks.jsonl` viewer (checker argv,
  exit, stdout hash, time) per gate.
- Conformance banner: runs workflow_lint (X3) per workspace; OFF-WORKFLOW
  red badge with the violated invariants listed.
- Deliverable viewer: embeds `output/deliverable/preview.html` (bundle tier)
  or PDF pages (existing); humanization rounds panel (rounds.json diff view).
- Pack/lock inspector: personalization.lock.json (hash-only) + resolved pack
  NAMES (never content) + floor warnings.
- ACTION MODE (`--allow-actions`, default OFF → viewer stays read-only):
  POST endpoints that shell out to pipeline_ctl only — run `check <gate>`,
  record human gate approval (writes APPROVALS.md line + `gate --mode
  supervised`), run content_audit, run doc_backend bundle. Localhost bind
  only; every action echoes the exact CLI it ran; no free-form command input.
- Events live tail (poll events.jsonl), dark theme, keyboard nav.

## X3 — Edit workflow + conformance enforcement

- `pipeline/references/stages-edit.yaml` — second stage graph for revising an
  EXISTING report: E0 intake (existing output + change request →
  edit_request.yaml, non-destructive copy) → E1 edit spec (human gate
  `edit_spec`) → E2 apply content edits (bundle/content.md deltas; sim numbers
  immutable unless the spec says re-run) → E2.5 content_audit (same checker) →
  E3 re-assemble (doc backend) → E3.5 verify_format + layout advisory →
  E4 final review (human gate `edit_accept`) → E5 deliver + archive diff.
- `pipeline_ctl init --graph build|edit` (default build): loads the named
  stages file; strict validation applies to both; `resume`/`check`/`gate`
  unchanged (graph-driven already).
- `pipeline/scripts/workflow_lint.py <WS>` — off-workflow detector, exit-coded:
  HARD: stage done past a script gate with no matching gate_checks.jsonl
  provenance; canonical_output set while gates pending; PIPELINE.md header
  edited outside pipeline_ctl (events.jsonl cross-check timestamps); delivered
  files in output/ newer than the last recorded assembly event.
  WARN: auto_approved human gates in supervised mode, stale heartbeat.
  Wired: stage-6 playbook step, Studio banner, and ORCHESTRATOR.md guardrail
  ("run workflow_lint before declaring done").
- Session-start conformance nudge: adapters/claude-code gets a hook snippet
  (documented, operator installs) that runs `pipeline_ctl resume` on session
  start and injects the result; ORCHESTRATOR.md states the first-action rule
  for harnesses without hooks.
